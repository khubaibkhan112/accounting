# Environment Variables Configuration

## Overview

This document describes all environment variables required for the Accounting Software application.

## Quick Setup

1. Copy `.env.example` to `.env`:
```bash
cp .env.example .env
```

2. Generate application key:
```bash
php artisan key:generate
```

3. Update the variables below with your actual values.

## Required Environment Variables

### Application Configuration

```env
APP_NAME="Accounting Software"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_TIMEZONE=UTC
APP_URL=http://localhost
APP_LOCALE=en
APP_FALLBACK_LOCALE=en
APP_FAKER_LOCALE=en_US
```

- **APP_NAME**: Name of your application
- **APP_ENV**: Environment (`local`, `staging`, `production`)
- **APP_KEY**: Application encryption key (generated by `php artisan key:generate`)
- **APP_DEBUG**: Enable/disable debug mode (`true` for development, `false` for production)
- **APP_TIMEZONE**: Application timezone
- **APP_URL**: Base URL of your application
- **APP_LOCALE**: Default locale
- **APP_FALLBACK_LOCALE**: Fallback locale
- **APP_FAKER_LOCALE**: Locale for faker data

### Database Configuration

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=accounting_db
DB_USERNAME=root
DB_PASSWORD=
```

- **DB_CONNECTION**: Database driver (`mysql`, `pgsql`, `sqlite`, `sqlsrv`)
- **DB_HOST**: Database host
- **DB_PORT**: Database port
- **DB_DATABASE**: Database name
- **DB_USERNAME**: Database username
- **DB_PASSWORD**: Database password

### Cache Configuration

```env
CACHE_STORE=database
CACHE_PREFIX=
```

- **CACHE_STORE**: Cache driver (`database`, `file`, `redis`, `memcached`)
- **CACHE_PREFIX**: Cache key prefix

### Session Configuration

```env
SESSION_DRIVER=database
SESSION_LIFETIME=120
SESSION_ENCRYPT=false
SESSION_PATH=/
SESSION_DOMAIN=null
```

- **SESSION_DRIVER**: Session driver (`database`, `file`, `redis`, `memcached`)
- **SESSION_LIFETIME**: Session lifetime in minutes
- **SESSION_ENCRYPT**: Encrypt session data
- **SESSION_PATH**: Session cookie path
- **SESSION_DOMAIN**: Session cookie domain

### Mail Configuration

```env
MAIL_MAILER=smtp
MAIL_HOST=mailpit
MAIL_PORT=1025
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@example.com"
MAIL_FROM_NAME="${APP_NAME}"
```

- **MAIL_MAILER**: Mail driver (`smtp`, `sendmail`, `mailpit`, `log`)
- **MAIL_HOST**: SMTP host
- **MAIL_PORT**: SMTP port
- **MAIL_USERNAME**: SMTP username
- **MAIL_PASSWORD**: SMTP password
- **MAIL_ENCRYPTION**: Encryption method (`tls`, `ssl`, `null`)
- **MAIL_FROM_ADDRESS**: Default sender email
- **MAIL_FROM_NAME**: Default sender name

### Queue Configuration

```env
QUEUE_CONNECTION=database
```

- **QUEUE_CONNECTION**: Queue driver (`database`, `redis`, `sync`)

### Logging Configuration

```env
LOG_CHANNEL=stack
LOG_STACK=single
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug
```

- **LOG_CHANNEL**: Log channel (`stack`, `single`, `daily`)
- **LOG_STACK**: Stack channel name
- **LOG_DEPRECATIONS_CHANNEL**: Deprecations log channel
- **LOG_LEVEL**: Log level (`debug`, `info`, `notice`, `warning`, `error`, `critical`, `alert`, `emergency`)

## Production Configuration

For production, ensure:

1. **APP_ENV=production**
2. **APP_DEBUG=false**
3. **APP_URL** set to your actual domain
4. Strong database credentials
5. Secure mail configuration
6. **SESSION_DRIVER=database** or **redis**
7. **CACHE_STORE=redis** or **database** (not `file` for production)

## Security Best Practices

1. Never commit `.env` file to version control
2. Use strong passwords for database
3. Set `APP_DEBUG=false` in production
4. Use HTTPS (set `APP_URL` with `https://`)
5. Use secure session and cache drivers
6. Regularly rotate `APP_KEY` if compromised


## Verification

After setting up environment variables, verify:

1. Application key is set:
```bash
php artisan key:generate
```

2. Database connection works:
```bash
php artisan migrate:status
```

3. Cache is working:
```bash
php artisan cache:clear
```

4. Configuration is cached (production):
```bash
php artisan config:cache
```
